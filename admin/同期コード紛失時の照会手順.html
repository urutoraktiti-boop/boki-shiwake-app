1. この手順書について

学生がスマホの機種変更やデータ削除の前に同期コード（BK2-XXXX-XXXX-XXXX）をメモし忘れて紛失した場合に、管理者（ULTRAK氏）が学生の大学名と学籍番号を手がかりに、その学生の同期コードを探し出すための手順です。

結論から言うと、同期コードそのものがFirestore上のドキュメントIDになっており、各ドキュメントには大学名（univ）と学籍番号（sid）が一緒に保存されています。だから、大学名・学籍番号で検索すれば、対応する同期コードを管理者が復元できます。
照会方法は2つあります。ふだんは方法A（管理者ダッシュボード）だけで足ります。マウス操作だけで、専門知識は不要です。方法Bのコンソールは、ダッシュボードで見つからないときの予備手段です。
方法難易度いつ使う
A管理者ダッシュボードやさしい（クリックのみ）基本はこちら。大学・学籍番号で探して行をクリックするだけ
BFirebaseコンソールふつうダッシュボードに出ない・細かく調べたいとき

2. 仕組み（なぜ照会できるのか）

アプリは学習データを Firestore の boki2_sync コレクションに保存しています。1人の学生＝1つのドキュメントで、中身は次のようになっています。

項目中身説明
ドキュメントIDBK2-7X2A-9QK4-M3PZ のような値これが同期コードそのもの。照会のゴールはこのIDを読むこと
univ例：日大商ホーム画面で登録した大学名
sid例：2400123ホーム画面で登録した学籍番号
updatedAt例：1751800000000最終同期日時（数値）。新しいほど直近
data学習履歴などの一式（長い文字列）成績データ本体。照会では見なくてよい
前提：学生が一度でも「大学名・学籍番号を登録」し、かつ「クラウド同期（コード発行）」を行っていることが必要です。どちらもしていない学生のデータはクラウド上に存在しないため照会できません（→5章）。

  • 方法Aで使うもの：管理者ダッシュボードのURL（下記）

4. 【方法A・推奨】管理者ダッシュボードで照会する

先生用の成績ダッシュボード（すでに稼働中）を使う、いちばんかんたんな方法です。クリック操作だけで同期コードが分かります。

  1. ダッシュボードを開いてログイン

    ブラウザで下記を開き、「Googleでログイン」から管理者アカウント（urutoraktiti@gmail.com）でログインします。

    https://urutoraktiti-boop.github.io/boki-shiwake-app/admin/

    ※管理者メール以外はログインしてもデータが表示されない設計です（安全）。

  2. 学生を絞り込む

    一覧が表示されたら、上部の「大学で絞り込み」で学生の大学を選ぶか、「学籍番号で検索」のボックスに学籍番号を入力して、対象の学生の行を探します。

    学籍番号での検索がいちばん速いです。大学が分かっていれば大学で絞ってから探してもOK。

  3. その学生の行をクリックする

    該当する行をクリックすると詳細がひらき、その中に 「（同期コード：BK2-XXXX-XXXX-XXXX）」 と表示されます。これが探している同期コードです。

    大学名・学籍番号も一緒に表示されるので、本人のものか必ず見比べてください。

  4. コードを控える

    表示された同期コードを正確に書き写します。0（ゼロ）と O（オー）、ハイフンの位置に注意してください。

    同じ学生が複数行あるとき：過去に何度もコードを発行していると同じ学生が複数出ることがあります。その場合は「最終更新」がいちばん新しい行を選んでください（＝直近まで使っていたデータ）。細かく判断したいときは方法B（コンソール）で updatedAt を直接比較します。
ダッシュボードで学生が見つからない場合は、まだクラウド同期をしていないか、大学・学籍番号の表記ゆれが原因のことが多いです。5章「見つからないときの対処」と、必要なら方法Bを確認してください。

5. 【方法B】Firebaseコンソールで照会する（予備）

方法Aで見つからないときや、updatedAt の値まで細かく確認したいときの手段です。Firebaseの管理画面を直接操作します。

  1. Firebaseコンソールを開く

    ブラウザで下記を開き、依頼者のGoogleアカウントでログインします。

    https://console.firebase.google.com/project/boki-shiwake-app/firestore

    ※上のURLで直接Firestoreが開きます。開かない場合は console.firebase.google.com → プロジェクト boki-shiwake-app → 左メニュー「構築 > Firestore Database」。

  2. コレクション boki2_sync を選ぶ

    「データ」タブの一覧から boki2_sync をクリックします。学生1人につき1つ、BK2-… という名前のドキュメントが並びます。

  3. 絞り込み（クエリ）で学籍番号を指定

    ドキュメント数が多いと目視では大変なので、フィルタを使います。ドキュメント一覧の上部にある「クエリを実行 / フィルタ」（漏斗・虫めがねのアイコン、または「クエリビルダー」）を開き、次の条件を追加します。

    フィールド条件値
    sid== （等しい）学生の学籍番号（例 2400123）
    まず学籍番号だけで絞るのがコツです。学籍番号はほぼ一意なので、たいてい1件に絞れます。複数大学で同じ学籍番号が衝突する場合のみ、univ == の条件も追加してください（2条件にすると索引作成を促されることがありますが、画面の案内に従えばOK）。

  4. ヒットしたドキュメントの「ID」を読む

    条件に一致したドキュメントが表示されます。そのドキュメント名（ID）＝BK2-XXXX-XXXX-XXXX が、探している同期コードです。念のため、中の univ と sid が学生本人と一致しているか確認します。

    複数ヒットした場合：同じ学生が過去に何度もコードを発行していると、複数のドキュメントが出ることがあります。updatedAt の数値がいちばん大きい（＝最新）ものを採用してください。それが直近まで使っていたデータです。

  5. コードを控える

    ドキュメントID（同期コード）を正確にコピーします。BK2- のハイフンや英数字の 0（ゼロ）と O（オー）などの読み違いに注意してください。

6. 見つからないときの対処

症状考えられる原因と対処
該当する sid が1件も出ない 学生がまだクラウド同期（コード発行）をしていない可能性が高い。この場合クラウドにデータが無いため復元不可。学生の端末にデータが残っていれば、その端末でコードを新規発行してもらう。
学籍番号は合っているが大学名が違う 登録時の表記ゆれ（「日大商」と「日本大学」など）。ダッシュボードなら大学の絞り込みを外して学籍番号だけで探す／コンソールなら univ 条件を外し sid だけで再検索する。
学籍番号を間違えて登録している sid の前後1桁違いなどを試す。大学名（univ ==）だけで絞り、一覧から本人を特定する方法も有効。
そもそもドキュメントが1件も無い 誰もまだ同期していない状態。学生に「初回にコード発行→メモ」を改めて案内する。
復元できないケース：同期コード未発行のまま端末のデータを消してしまった学習記録は、クラウドにも端末にも存在しないため復元できません。これを防ぐため、学生には初回のコード発行とメモ保管を必ず案内してください。

7. 学生へのコードの伝え方

  • コードを知っていれば誰でもそのデータを読める設計です。本人確認のうえ、口頭・DM・メールなど本人だけに届く手段で伝えてください。
  • 学生は新しい端末でアプリを開き、「クラウド同期」→「復元」から受け取ったコードを入力すれば、成績データが元通りになります。
  • 復元後はそのコードを大切にメモしておくよう改めて念押しします。

8. 上級者向け：URLで直接照会する方法

コンソールを使わず、コードが分かっている場合にデータの存在だけ確認したいときは、ブラウザのアドレスバーに次を貼るとJSONが表示されます（<CODE> を実際のコードに置換）。

https://firestore.googleapis.com/v1/projects/boki-shiwake-app/databases/(default)/documents/boki2_sync/<CODE>?key=AIzaSyC29p0TS5K8FFs4s0ICxRrcOSLdXOnwIII
この方法はコードが既に分かっているときの確認用です。セキュリティルールで一覧取得（list）は禁止しているため、コードを知らない状態で大学名・学籍番号から逆引きすることはREST APIではできません。逆引きは必ず方法A（ダッシュボード）か方法B（コンソール）の管理者権限で行ってください。

9. セキュリティ・運用上の注意

  • この手順書にはFirebaseプロジェクト名・APIキーが含まれます。管理者以外に共有しないでください（※APIキー自体はアプリのソースにも含まれる公開値ですが、手順書は管理者用として扱う方針）。
  • 照会作業は管理者本人のGoogleアカウント（urutoraktiti@gmail.com）でのみ行い、ログイン情報は他者に渡さないこと。
  • 同期コードは秘匿が前提の設計です。照会したコードを掲示板・共有ドキュメント等に残さないでください。
  • ダッシュボードで全件を一覧できるのは、Firebase Authの管理者ログイン＋Firestoreルールの list 制限（管理者メール限定）で保護されているためです。管理者を増やすときは、firestore.rules と admin/index.html の ADMIN_EMAILS の両方にメールを追記します（引き継ぎ書 §5 参照）。
