# 管理者ダッシュボード セットアップ手順

先生が「大学別・学籍番号別の成績一覧」を見るためのページ（`admin/index.html`）の
準備手順です。**初回だけ、Firebaseコンソールで4つの設定**を行えば、あとはページを
開いてログインするだけで使えます。

- ダッシュボードのURL（設置後）：
  **https://urutoraktiti-boop.github.io/boki-shiwake-app/admin/**

> 🔒 このページは公開URLにありますが、成績データを取得できるのは
> 下で設定する**管理者のGoogleアカウントでログインした人だけ**です
> （Firestoreのセキュリティルールで制御しているため、他人が開いても中身は見えません）。

---

## 初回セットアップ（Firebaseコンソール）

Firebaseコンソール → プロジェクト **boki-shiwake-app** を開いて、順に行います。

### ① Googleログインを有効化する
1. 左メニュー **Authentication（構築 > Authentication）** を開く
2. 「始める」→ **Sign-in method** タブ
3. 「新しいプロバイダを追加」→ **Google** を選ぶ → **有効にする** → 保存
   - サポートメールは自分のGmailを選べばOK

### ② 公開ドメインを許可リストに追加する
1. **Authentication** →（上部）**Settings** → **承認済みドメイン（Authorized domains）**
2. 「ドメインを追加」で **`urutoraktiti-boop.github.io`** を追加
   - これが無いと、ログインのポップアップが弾かれます
   - `localhost` は最初から入っていることが多い（手元確認用）

### ③ セキュリティルールを貼り替える
1. 左メニュー **Firestore Database** → **ルール** タブ
2. リポジトリの **`firestore.rules`** の中身をすべてコピーして貼り付け
3. **公開（Publish）** を押す
   - これで「管理者メールのときだけ一覧できる」ようになります
   - 学生側は今までどおり同期コードだけで使えます（影響なし）

### ④ 管理者メールが一致しているか確認する
以下の2か所のメールアドレスが**同じ**であること：
- `firestore.rules` の `'urutoraktiti@gmail.com'`
- `admin/index.html` の `ADMIN_EMAILS = ["urutoraktiti@gmail.com"]`

> 管理者を増やしたいときは、**両方**の配列にメールを追記してください。
> 例）`['urutoraktiti@gmail.com', 'sensei2@example.com']`

## 使い方

1. ダッシュボードURLを開く
2. **「Googleでログイン」** → 管理者アカウントを選ぶ
3. 成績一覧が表示される
   - 上部に「登録学生数・大学数・受験実績・直近7日の活動」のサマリー
   - 表は **大学別の開閉式**。大学行をクリックすると、その大学の学生一覧が開く
   - 学生行は **大学 / 学籍番号 × 回別テスト①〜⑨** の最高点（70%以上は緑）
   - **大学で絞り込み**、**学籍番号で検索**、**列見出しクリックで並べ替え**
   - **学生行をクリック**すると、その学生の各テストの詳細（最高点・挑戦回数・直近日）
   - **CSV書き出し**でExcelに取り込み可能
   - 「課題の作成・管理」の各課題にある **成績結果** ボタンを押すと、提出率・平均点・平均所要時間・問題別正答率・学生別結果を確認できる

## よくあるトラブル

| 症状 | 原因・対処 |
|---|---|
| ログインのポップアップが出ない/エラー | ②の承認済みドメインに `urutoraktiti-boop.github.io` を追加したか確認 |
| 「閲覧権限がありません」と出る | ④のメール不一致。ログインしたGoogleアカウントのメールが `ADMIN_EMAILS`・ルール両方に入っているか確認 |
| 「取得エラー（permission-denied）」 | ③のルール公開を忘れていないか、メールが一致しているか確認 |
| 学生が表に出てこない | その学生がまだ「同期コードを発行」していない＝クラウドに保存されていない。学生に初回のコード発行を案内 |
| 大学・学籍番号が「（未登録）」 | 学生がホームの「ユーザー情報」を保存していない。学生に登録を案内 |

## しくみ（技術メモ・次のエージェント向け）

- 認証：Firebase Authentication（Googleプロバイダ）。ダッシュボードのみ Firebase JS SDK（v10, CDN）を使用
  （学生用アプリは従来どおり Firestore REST・SDK不使用のまま）
- データ取得：`getDocs(collection(db,"boki2_sync"))` = **list** 操作。ルールで管理者メールに限定
- 各ドキュメントのフィールド：`univ` / `sid` / `updatedAt` / `data`（学習記録のJSON文字列）
- 成績集計：`data` を JSON.parse し、`tests[]`（`{group,score,max,correct,total,date}`）から
  回別テストごとの**最高点・挑戦回数・直近日**を算出。合計は各回の最高点の総和
- 料金：Authentication・Firestore読み取りとも無料(Spark)プランの範囲内（学生数十名規模）
