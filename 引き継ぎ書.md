## 1. プロジェクトの目的と背景

日商簿記2級の合格を目指す大学生向けに、仕訳問題（第1問対策）をPC・タブレット・スマホで反復学習できるアプリを開発・配布するプロジェクト。依頼者（ULTRAK氏）は大学生の学習指導をしており、教材PDF（問題＋解答解説）をアプリ化して学生に配布し、大学別・学籍番号別に成績を管理者が把握できる形を目指している。

依頼者は開発者ではないが、別アプリ「コツコツサクサクアプリ」をGitHub配布＋Firebase同期コードで運用した経験がある。説明は初心者向けに丁寧に、常に日本語で行うこと。

## 4. 学生用アプリの機能仕様（v3.21時点）

単一HTMLファイル。データ保存は localStorage（キー `boki2_shiwake_20260623`）＋Firebase同期。

- **ホーム（v3.18で管理者確認モードを追加）**: ①回答テスト①〜⑨のメインボタン（直近点数表示、70%以上で緑）②その他の練習（全問・復習・ランダム・分野出題）③⚙ 設定：**💾 データ保存・引き継ぎ**（開閉式。中に**👤 ユーザー情報**、**☁ クラウド同期**、最下部に「📖 データを守る・機種変更で引き継ぐ方法」ガイドリンクを配置。ボタン文言に登録済み/未登録・同期有効/未設定を表示）。ユーザー情報は登録済みの場合、現在の登録情報を表示し「ユーザー情報を修正する」ボタンを押したときだけ入力欄を表示。保存しても同期コードは変わらず、次回同期後に管理者画面の大学名・学籍番号へ反映される／**🗃 問題セット・分野の管理**（＝配布版では非表示。`BUNDLED_SETS.length>0` のとき `manage-wrap` を隠す実装。マスター版のみ表示）。アプリ最下部に小さな「管理者ダッシュボード」リンクを設置し、`admin/` を別タブで開く
  - **管理者確認モード（v3.18）**: URLに `?preview=admin` を付けると、初回登録ポップアップを出さずに学生画面を確認できる。管理者ダッシュボードの大学別告知リンクでは、学生へコピーするURLは通常どおり `?univ=...` のまま、画面確認用の「確認用に開く」だけ `?univ=...&preview=admin` を開く。確認モードでは端末内にプロフィールや同期コードが残っていてもクラウド保存を止めるため、解いた記録は管理者ダッシュボード集計には上がらない
  - **大学別URL / QRコード（v3.16）**: 学生用URLに `?univ=gakushuin` のような大学コードを付けると、その大学専用リンクとして扱う。初回登録ポップアップでは大学名を選ばせず「所属大学：学習院大学」と固定表示し、学生は学籍番号だけ入力する。設定内の修正画面でも大学名はそのリンクの大学に固定される。通常URL（`?univ=`なし）では従来どおり大学選択＋その他入力が使える。対応表は `UNIVERSITY_LINKS`（学生アプリ）と管理画面の同名定数で管理。現時点のコード例：`gakushuin` / `nichidai-econ` / `nichidai-commerce` / `toyo` / `komazawa` / `senshu`
  - **初回起動時の必須登録（v3.7、v3.14で修正導線を明確化、v3.16で大学固定リンク対応）**: 未登録なら `requireProfile()` が閉じられないポップアップで大学・学籍番号の入力を促す。大学別URLの場合は大学名を固定し、学籍番号だけを入力させる（`commitProfile()` で保存）。以後は「設定 → データ保存・引き継ぎ → ユーザー情報を修正する」から変更可
  - **登録時に同期コードを自動発行＝全員が自動でダッシュボードに出る（v3.12・重要）**: 以前は生徒が自分で「同期コードを発行」を押さないとクラウド（Firestore）に上がらず、**発行した生徒しか管理者ダッシュボードに出なかった**（`cloudSave()` は `store.sync.code` が無いと何もしない仕様）。v3.12で `commitProfile()` と起動処理（`initSync()` 後）に `ensureSyncCode()` を追加：**大学・学籍番号を登録済みで同期コードが無ければ自動でコード発行＋クラウド保存**する。これにより新規生徒は登録だけでダッシュボードに反映。**既存の登録済み・コード未発行の生徒も、更新後にアプリを一度開けば自動アップロードされて出る**（開かない生徒の記録は端末内にのみ残り、クラウドからは取得不可）。機種変更用の同期コードは v3.13 以降、設定→データ保存・引き継ぎ で確認・コピー可
  - **初回ポップアップからの同期コード復元（v3.8）**: 上記ポップアップ下部に「すでに同期コードをお持ちの方」の入力欄＋「復元」ボタンを追加。機種変更した学生が新端末の初回画面からそのまま記録を引き継げる。復元処理は `restoreByCode(code)` に切り出し（`enterSyncCode()` のプロンプト版もこれを呼ぶ）。復元成功かつ profile が復元されればポップアップを閉じる
- **問選択ポップアップ**: テスト①〜⑨を押すと問1〜5を複数選択して出題（未選択の問は「未解答」のまま。配点も選択分の合計満点 ＝ ダッシュボードで分母が人により変わる理由）
- **管理者からの課題（自動配信・v3.10）**: 管理者がダッシュボードで作った課題（①〜⑨から選んだ問のセット）が、対象（全員／大学／学籍番号）に合致する生徒のホーム上部に「📌 管理者からの課題」カードとして**自動で表示**（コード入力不要）。押すと採点付きで出題し、結果は同期でダッシュボードに集約。実装：起動時とプロフィール保存時に `loadAssignments()`（`boki2_sync/assignments-config` をRESTでGET・**読むだけ**）→ `myAssignments()`（`store.profile` で対象判定）→ `renderAssignments()`。出題は `startAssignment(id)`（`startPicked` を雛形に qid配列から採点セッション、`groupName="課題:"+title`, `assignmentId` を保持）。結果は `finish()` が `store.tests` に `assignmentId` と問別⭕❌の `detail[]` を付けて記録
- **問題公開設定（v3.21）**: `boki2_sync/publications-config` をRESTで読み、対象（全員／大学／大学＋学籍番号）と公開開始・終了日時が一致する問題だけを表示する。2025年度直前模試48問は `CONTROLLED_QIDS` で初期状態から管理対象とし、設定が未取得・未作成でも非表示（安全側）。従来49問は互換性のため従来どおり表示する。設定はlocalStorageへキャッシュし、開始・終了時刻に画面を自動更新する
- **解答画面**: 記号は10文字幅プルダウン、金額はカンマ込み8桁幅＋datalist候補＋直接入力。都度採点もまとめ採点も可。採点後の解説は文字を必須とし、`explanationImages` がある場合はその下に確認済みPNGとキャプションを表示（v3.19）。画像パスは `assets/explanations/` 配下のPNGだけを許可し、未確認画像は表示しない。全問後に合計点（20点満点、仕訳4点・枝問2点）と⭕❌明細。**画面下部に「■ 中断してホームに戻る」ボタン（v3.6）**：押すと3択ポップアップ（保存して中断／保存せずにやめる／キャンセル）。「保存して中断」で解答中セッションを保持しホームの「▶ 解答中の問題に戻る」から再開可（※実装は `confirmQuit()`／`session` はメモリ保持のみ。**アプリを閉じる・再読み込みすると途中の答えは消える**仕様。「保存せずにやめる」は `session=null`）
- **解答ストップウォッチ（v3.9）**: 各問題の所要時間を計測。問題を開くと画面上部（進捗表示の隣）に `⏱ mm:ss` のストップウォッチが表示され経過時間をカウント。**採点した時点でその問題の時間を確定（凍結）**、次の問題へ進むと次問はゼロから再計測。全問終了後のサマリーに **トータル所要時間** と各問題の所要時間を表示。**別タブ・バックグラウンドに離れている間はタイマー一時停止**（`visibilitychange` で `pauseTimer`／復帰で `beginTimer`）。中断→キャンセルや「解答中の問題に戻る」でも再開する。実装は `beginTimer`／`pauseTimer`／`swDisplay`／`fmtClock`／`fmtDur`／`fmtDiff`（`session.times[]`/`tFrozen[]`/`tStart`/`tIdx` に計測状態を保持。`session` はメモリ保持のみのため再読み込みで消える点は従来同様）
- **成績**: サマリー → 回別テスト成績（棒グラフ・前回比／**総所要時間と前回比（±分秒・速いほど緑＝改善）も表示 v3.9**）→ 案Bヒートマップ → 案C分野別成長比較 → 日別チャート → 分野別/問題別（**問題別に平均所要時間を併記 v3.9**）→ 成績レポート保存＋印刷/PDF
- **問題データ形式（JSON）**: `{name, questions:[{label(必須), category, topic, text, options{ア..ク}, debit[[記号,金額]..], credit[..], expl, explanationImages?, group, points}]}`。`topic` は `skills/extract-boki-mock-pdf/references/title-map.json` の固定タイトルを使用する。`explanationImages` は確認済みPNGの `src`、`alt`、`caption`、出典ページ、切り抜き座標、寸法、SHA-256等を持つ配列（画像不要なら `[]`）。取込時に貸借一致・記号整合を自動検証。IDは `セット名_qN` で自動採番（**セット名を変えると履歴が切れる**点に注意）
- **同期ドキュメント構造**（`boki2_sync/{同期コード}`）:
  - `univ`（string）/ `sid`（string）/ `updatedAt`（integer）/ `data`（学習記録のJSON文字列）
  - `data` の中身 = `{history, sets, meta, tests, profile, updatedAt}`
  - `history` = `[{qid, correct, date, ms?}]`（`ms`＝その解答の所要時間ミリ秒。**v3.9で追加**。旧レコードには無いので表示側は `ms!=null` を確認）
  - `tests` = `[{group, score, max, correct, total, date, timeMs?, assignmentId?, detail?}]`（回別テストの挑戦記録。ダッシュボードはこれを集計。`timeMs`＝そのテストの総所要時間ミリ秒（**v3.9**）。`assignmentId`＝管理者課題の識別ID、`detail`＝`[{qid,label,correct,points}]` の問別⭕❌（**v3.10で課題用に追加**。課題の結果はメイン表①〜⑨には出さず「課題の結果」に集約）。旧レコードには無い）
  - **課題定義ドキュメント**（`boki2_sync/assignments-config`）: 管理者が書き（生徒は読むだけ）。中身 `data`（JSON文字列）= `{assignments:[{id,title,target,qids,createdAt}], updatedAt}`。`target`＝`{mode:"all"}`／`{mode:"univ",univ}`／`{mode:"sids",sids:[]}`。大学別管理者は画面上、担当大学向けの課題のみ作成・編集・結果確認できる。`qids` は `第1問特集①〜⑨_qN` 形式。`assignments-config` の `create/update` はFirestoreルールで管理者限定（生徒アプリからの `get` は従来どおり全員許可）。ダッシュボードは `list` 取得時にこの `doc.id`（= `ASSIGN_DOC`）を学生行から除外する。⚠️**ドキュメントIDに `__..__`（前後アンダースコア2つ）は使えない**（Firestoreの予約ID。v1.3までは `__assignments__` で保存エラーになっていたため v1.4 で `assignments-config` に変更）
  - **問題公開設定ドキュメント**（`boki2_sync/publications-config`、v3.21／管理画面v1.13）: `data`（JSON文字列）= `{publications:[{id,title,ownerEmail,target,qids,enabled,publishFrom,publishUntil,createdAt,updatedAt}],updatedAt}`。大学別管理者は自分が所有し、担当大学を対象とする設定だけを編集できる。保存・削除はFirestore transactionで競合を抑える。学生は読むだけ

## 8. 既知の注意点・リスク

- 学習記録はブラウザ単位。同期コード未発行のままデータ消去すると復元不可（学生には初回のコード発行とメモを案内する前提）。**v3.12以降は登録時に同期コードを自動発行するため、原則として登録＝クラウド保存される**（コードは v3.13 以降、設定→データ保存・引き継ぎ で確認・メモ可）
- **【重要・v3.12以前の既知の落とし穴】管理者ダッシュボードに出るのは「クラウドに上がった生徒」だけ**。v3.11以前は生徒が手動で同期コードを発行しないとアップロードされず、解答した人数より登録者数が大幅に少なくなる事象が起きた（例：100名受験→8名のみ表示）。v3.12の自動発行で解消。ただし**過去に旧版で解いた生徒の記録は、その生徒が新版を一度開くまでクラウドに来ない**点は変わらない（端末内には残っている）
- **解答中の「中断→保存して中断」で保持されるのはメモリ上のセッションのみ（v3.6）**。同じ画面を開いている間は「解答中の問題に戻る」で再開できるが、タブを閉じる・再読み込みすると途中の（未採点の）答えは失われる。将来「後日も続きから」に対応したい場合は `session` を localStorage に一時保存する実装追加が必要（依頼者の選択で今回はメモリ保持のみに決定）
- **1問ごとに「採点する」を押した時点で正誤が履歴に即保存される**。そのため「保存せずにやめる（破棄）」を選んでも、すでに採点済みの問題は分野別・問題別・復習の集計に残る（回別テストの合計点は「全体の結果へ」到達時のみ記録なので、中断すれば中途半端な点数は残らない）
- 同期コードを知っていれば誰でもそのデータを**読める**設計（コードの秘匿が前提。既存アプリと同水準）。※一覧（list）は管理者のみ
- 教材由来の問題文・必要図解について、**教材権利者から本アプリでの利用許可を取得済み**（2026-07-10、依頼者が再確認）。PDF原本や全ページ画像は公開リポジトリへ置かず、アプリ用に確認済みの問題データと必要箇所だけの切り抜き画像を扱う
- **問題公開設定は画面上の表示制御**。問題本文が配布版HTMLや端末内データに含まれる構成では、開発者ツール等による完全な秘匿・公開後のオフライン端末からの回収はできない。厳密な機密配信が必要なら、問題本文自体を認証付きAPIから公開時に取得する構成へ変更する
- 成績はブラウザ→Firestoreの自己申告ベース（改ざん耐性なし）。割り切り
- GitHub Pages上のアプリとローカルで開いたHTMLはlocalStorageが別（同期コードで引継ぎ可能）
- ダッシュボードの①〜⑨列の分母は学生ごとに変わりうる（問選択ポップアップで受験した問だけが満点になるため。仕様どおり）
- 管理者ページは公開URL上にあるが、データ取得はログイン＋ルールで保護されるため他人には中身が見えない
