# 管理者ダッシュボード セットアップ手順

先生が「大学別・学籍番号別の成績一覧」を見るためのページ（`admin/index.html`）の
準備手順です。**初回だけ、Firebaseコンソールで4つの設定**を行えば、あとはページを
開いてログインするだけで使えます。

- ダッシュボードのURL（設置後）：
  **https://urutoraktiti-boop.github.io/boki-shiwake-app/admin/**

> 🔒 このページは公開URLにありますが、成績データを取得できるのは
> 下で設定する**管理者アカウントでログインした人だけ**です
> （Firestoreのセキュリティルールで制御しているため、他人が開いても中身は見えません）。

---

## 初回セットアップ（Firebaseコンソール）

Firebaseコンソール → プロジェクト **boki-shiwake-app** を開いて、順に行います。

### ① ログイン方法を有効化する
1. 左メニュー **Authentication（構築 > Authentication）** を開く
2. 「始める」→ **Sign-in method** タブ
3. 「新しいプロバイダを追加」→ **Google** を選ぶ → **有効にする** → 保存
   - サポートメールは自分のGmailを選べばOK
4. 同じく **メール/パスワード** を選ぶ → **有効にする** → 保存
   - これでGoogle以外の大学メールでも、メールアドレス＋パスワードでログインできます
5. **Templates（テンプレート）** タブで、テンプレート言語が **日本語** になっていることを確認
   - 2026-07-09時点で、Firebase標準のパスワード設定メールは日本語表示に設定済み
   - Sparkプランの標準テンプレートは個別文面の自由編集ができない場合があります

### ② 公開ドメインを許可リストに追加する
1. **Authentication** →（上部）**Settings** → **承認済みドメイン（Authorized domains）**
2. 「ドメインを追加」で **`urutoraktiti-boop.github.io`** を追加
   - これが無いと、ログインのポップアップが弾かれます
   - `localhost` は最初から入っていることが多い（手元確認用）

### ③ セキュリティルールを貼り替える
1. 左メニュー **Firestore Database** → **ルール** タブ
2. リポジトリの **`firestore.rules`** の中身をすべてコピーして貼り付け
3. **公開（Publish）** を押す
   - これで「マスター管理者、または管理画面で登録した管理者だけ一覧できる」ようになります
   - 学生側は今までどおり同期コードだけで使えます（影響なし）

### ④ マスター管理者メールが一致しているか確認する
以下の2か所のメールアドレスが**同じ**であること：
- `firestore.rules` の `isMasterAdmin()` 内の `'urutoraktiti@gmail.com'`
- `admin/index.html` の `MASTER_ADMIN_EMAILS = ["urutoraktiti@gmail.com"]`

> 通常の大学別管理者は、コード編集ではなく管理画面の「管理者権限設定」から追加します。

## 使い方

1. ダッシュボードURLを開く
2. ログイン方法を選ぶ
   - Googleを使える場合：**「Googleでログイン」** → 管理者アカウントを選ぶ
   - Google以外のメールを使う場合：メールアドレスとパスワードでログイン
3. 成績一覧が表示される
   - 上部に「登録学生数・大学数・受験実績・直近7日の活動」のサマリー
   - **大学別の告知リンク** から、各大学専用の学生用URLをコピーできます
     - 例：学習院大学用URLをQRコードにすると、学生側では大学名が固定され、学籍番号だけ入力する形になります
   - マスター管理者だけ、**管理者権限設定** からメールアドレス・権限・担当大学を登録できます
   - 表は **大学別の開閉式**。大学行をクリックすると、その大学の学生一覧が開く
   - 学生行は **大学 / 学籍番号 × 回別テスト①〜⑨** の最高点（70%以上は緑）
   - **大学で絞り込み**、**学籍番号で検索**、**列見出しクリックで並べ替え**
   - **学生行をクリック**すると、その学生の各テストの詳細（最高点・挑戦回数・直近日）
   - **CSV書き出し**でExcelに取り込み可能
   - 「課題の作成・管理」の各課題にある **成績結果** ボタンを押すと、提出率・平均点・平均所要時間・問題別正答率・学生別結果を確認できる

### 管理者を追加する

1. マスター管理者（`urutoraktiti@gmail.com`）でログイン
2. **管理者権限設定** にメールアドレスを入力
3. 権限を選ぶ
   - **大学別管理者**：担当大学の学生・課題だけ表示
   - **マスター管理者**：全大学を表示、管理者権限設定も編集可
4. 大学別管理者の場合は担当大学を選ぶ
5. **追加・更新** を押す
6. 確認リンクが作成されるので、追加された管理者へ送る
7. 管理者本人が確認リンクを開く
8. メールアドレス・権限・担当大学が正しいか確認する
9. 正しければ **内容を確認してパスワード設定メールを送る** を押す
10. Firebaseから届くパスワード設定メールでパスワードを設定する
11. メールアドレス＋パスワードでログインする

> 確認リンクを作り直す場合は、管理者一覧の **確認リンク作成** を押してください。

### メールアドレス＋パスワードでログインする

1. 管理者ダッシュボードを開く
2. メールアドレスとパスワードを入力
3. **ログイン** を押す
4. 以後は同じメールアドレス＋パスワードでログイン
5. パスワードを忘れた場合は **パスワード再設定** を押す

> 旧運用・再発行用として、ログイン画面には **初回パスワード設定** も残しています。

> Firebase標準メールの文面には「パスワードを再設定」という表現が残ります。現状は、最初に独自の確認リンクで担当大学を確認し、確認後にFirebase標準のパスワード設定メールを送る運用です。

## よくあるトラブル

| 症状 | 原因・対処 |
|---|---|
| ログインのポップアップが出ない/エラー | Googleログインの場合は、②の承認済みドメインに `urutoraktiti-boop.github.io` を追加したか確認 |
| メール/パスワードでエラーになる | ①で **メール/パスワード** プロバイダを有効化しているか確認 |
| 確認リンクが分からない | マスター管理者が管理者一覧の **確認リンク作成** を押し、リンクを送る |
| 確認画面の担当大学が違う | パスワード設定メールを送らず、マスター管理者に連絡。マスター管理者が担当大学を直して **確認リンク作成** を押す |
| パスワード設定メールが届かない | 迷惑メールを確認。届かない場合は確認リンクを開き直して、もう一度 **内容を確認してパスワード設定メールを送る** を押す |
| 「閲覧権限がありません」と出る | 管理者権限設定に登録されていない、または別のメールアドレスでログインしている可能性があります |
| 「取得エラー（permission-denied）」 | ③のルール公開を忘れていないか、マスター管理者メールと管理者権限設定を確認 |
| 学生が表に出てこない | 学生がまだ初回登録をしていない、または最新版を開いてクラウド同期していない可能性があります。学生に大学別QRコードから開き、学籍番号を登録してもらう |
| 大学・学籍番号が「（未登録）」 | 学生がホームの「ユーザー情報」を保存していない。大学別QRコードから開いて学籍番号を登録するよう案内 |
| 学生が別の大学で登録されている | 正しい大学別URL / QRコードから開き、「設定 → データ保存・引き継ぎ → ユーザー情報を修正する」で保存し直す |

## しくみ（技術メモ・次のエージェント向け）

- 認証：Firebase Authentication（Googleプロバイダ＋メール/パスワード）。ダッシュボードのみ Firebase JS SDK（v10, CDN）を使用
  （学生用アプリは従来どおり Firestore REST・SDK不使用のまま）
- データ取得：`getDocs(collection(db,"boki2_sync"))` = **list** 操作。ルールでマスター管理者または `admin-config` 登録済み管理者に限定
- 各ドキュメントのフィールド：`univ` / `sid` / `updatedAt` / `data`（学習記録のJSON文字列）
- 大学別告知リンク：`UNIVERSITY_LINKS` の `code` を学生用URLの `?univ=` に付ける。学生アプリ側にも同じ `UNIVERSITY_LINKS` があり、大学名固定表示に使う
- 管理者権限設定：`boki2_sync/admin-config` に `data`（JSON）と `adminsMap` を保存。Firestoreルールは `adminsMap` を参照して登録済み管理者の `list` を許可する。管理者追加後は確認リンクを発行し、確認ページ側からFirebase標準のパスワード設定メールを送る
- 成績集計：`data` を JSON.parse し、`tests[]`（`{group,score,max,correct,total,date}`）から
  回別テストごとの**最高点・挑戦回数・直近日**を算出。合計は各回の最高点の総和
- 料金：Authentication・Firestore読み取りとも無料(Spark)プランの範囲内（学生数十名規模）
